## A. Overview and honest expected-outcome statement

### A.1 What is being tested

Back the **underdog** — the side with the higher decimal win price / lower implied win probability (CALC §2.1) — in FIFA men's World Cup matches via **Draw-No-Bet** (stake refunded on a 90-minute draw; equivalent to Asian Handicap 0.0). DNB is either the directly quoted `AHh = 0` / `AHCh = 0` price where it exists, or synthesised from 1X2 via the key identity `o_DNB = W·(D−1)/D` (CALC §3, DATA §1.2, ARCH §1.3). Settlement is on the **90-minute result** — knockout extra time and penalties never change the bet outcome (CALC §10, DATA §7.2, EDGE §4.6).

Because the World-Cup-only sample (~384 clean-odds matches 2002–2022; 104 matches in 2026) is statistically underpowered (STAT §6.3; DATA §3.3 — even a +15.5% per-bet edge is the smallest detectable on 384 bets), the estimation universe is **expanded to domestic leagues** (football-data.co.uk Pinnacle closing 1X2 + Asian-Handicap, season-conditional reference price per DATA §2.1), and the World Cup is the **held-out, never-touched test block** (DATA §4, STAT §9.1, methodology.md §2). The rule and every hyperparameter are frozen on the league universe by walk-forward CV; the World Cup is scored once.

### A.2 Honest expected outcome (this is a quantification, not a profit promise)

The prior is a **wide confidence interval around zero, most plausibly net-negative EV.** This is the load-bearing honesty statement the deliverable must defend rather than contradict:

1. **The 1X2 favourite-longshot bias makes the naive underdog bet a loser** (~−9% to −10% per win bet for longshots vs −6% for favourites; EDGE §1.3, §3.2; Cain–Law–Peel 2000; Snowberg–Wolfers 2010). The synthetic DNB carries the **full 1X2 margin on both legs** (CALC §3.5, §8.2 worked −5.3% EV; EDGE §5.3 −5% to −7% under the primary Shin de-vig), so the synthetic route is structurally negative-EV.
2. **The one piece of evidence pointing the project's way is narrow and small.** In the Asian-Handicap / DNB family the FLB is absent-to-weakly-reverse and the underdog is the *cheaper* side, but the instrument-correct loss prior is the integer/full-refund figure ≈ **3.24%** (market-average; EDGE §3.3, §3.5; Hegarty–Whelan 2024 RBF Table 2), **not** the 3.28% weak-team cell, which explicitly excludes the zero handicap. The favourable cross-sectional gradient (~0.9 pp) is **smaller than the margin itself**, and only the price-based gradient is candidate-transferable to neutral World-Cup venues; neutral-venue probability compression is expected to *shrink* it further (EDGE §3.5, §4.2, §7).
3. **Power is the binding constraint.** A plausible single-digit-percent edge needs ~10³–10⁴ bets at 80% power (STAT §6.3; DATA §3.3); the World Cup hold-out alone cannot resolve it. The honest deliverable therefore **bounds the loss and tests reverse-FLB transfer**, and reports the bankroll/`λ` that *would* be required were the edge real (a feasibility/power statement), rather than asserting a positive edge (STAKE §7.3, §8; EDGE §7).
4. **The most likely honest verdict is `λ* = 0` (do not bet).** If the White/Hansen-corrected, deflated analysis cannot reject the no-edge null, that is the pre-committed output — the multiple-testing register exists precisely to prevent reverse-engineering a surviving cell (STAKE Open Question 6; STAT §7; methodology.md §1.2). This verdict is defended on **net-of-cost, executable terms**: every headline ROI/Sharpe is reported after the transaction-cost / execution model (per-leg slippage, the two-leg leg-out idealization stress-tested by a one-tick adverse move, and the BFEC commission as an effective overround), so the no-edge conclusion rests on the tradable instrument, not the idealized synthetic one (Phase 0 ADR 7.1; Phase 3 task 9; gaps register transaction-cost row).

The deliverable's value is a rigorously bounded, selection-bias-corrected, reproducible **estimate of the edge (or its absence)** with confidence intervals — and an explicit statement of what would have had to be true for EV > 0 (EDGE §5.4).

**Methods (cited).**

  7. `snapshot_date` — vendor snapshot date
  8. `rng_seed` — the root seed (see "Determinism guarantees" below) *(CLAUDE.md-mandated: RNG seed)*

  12. `timestamp` — ISO-8601 emission time
  13. `runtime` — object capturing `{python_version, platform}` (the interpreter + OS/arch fingerprint, kept as one composite key so the total is exactly 13)

### Phase 2 — DNB construction, underdog labelling, 90-minute settlement

**Objective.** Implement and unit-prove the load-bearing pricing/labelling/settlement core: synthetic DNB from 1X2, de-vig to fair probabilities, underdog selection, and the three-outcome 90-minute settlement map.

**Inputs.** `data/processed/matches.parquet` (Phase 1); the frozen `design.md` (Phase 0); CALC, DATA, ARCH pricing/settlement contracts.

**Methods (cited).**
- Synthetic DNB identity and its no-arbitrage replication proof: **CALC §3.1–§3.3** (`o_DNB = W·(D−1)/D = W·(1−r_D)`); equivalence to AH-0.0 and the conditional 1X2: **CALC §3.4**; reconciliation with the quoted AH-0 price and the margin wedge: **CALC §3.5**. - De-vig to fair `(p_W, p_D, p_fav)`: **CALC §4**. The de-vig method is **frozen a priori by Phase-0 pre-registration — Shin is the primary; power and basic are registered sensitivities — and is therefore *not* one of the per-fold-selected hyperparameters** (Phase 0 task 5 / task 6; CALC §4.6). ### Phase 3 — Staking grid and bankroll / risk-of-ruin simulation

**Objective.** Implement the staking schemes (flat / fixed-fraction / level-to-odds / Kelly / fractional-Kelly), the concurrent-matchday vector Kelly, and the matchday-block bootstrap risk-of-ruin / minimum-bankroll / growth–drawdown frontier — all as the *contract* the multiple-testing gate (Phase 4) consumes.

**Inputs.** `PricedBets` (Phase 2: `o_DNB`, de-vigged `p,d`); the risk decision inputs `(ρ, α_dd, β_dd)` grids from methodology.md §1.2; STAKE; STAT §2.5–§2.6.

**Methods (cited).**
- Staking schemes and what the grid holds fixed vs varies: **STAKE §2, §7.1**. Fixed-fraction `φ` is the canonical scale-free "flat" comparator; level-to-odds stakes `∝ 1/(d−1)`; fractional Kelly `λ·f*` — the scientific question is whether endogenous edge-proportional (Kelly) sizing beats odds-agnostic and naively-tilted sizing on drawdown-aware growth (STAKE §7.1).
- Three-outcome push-Kelly: **STAKE §3** (`f* = (p_A·b − p_H)/[b(p_A+p_H)]`, the binary Kelly on no-push-renormalised probabilities); bet only if `p̃_A·d_A > 1`; negative-edge → `f* = 0` (no short side). Mirrors CALC §7.
- Fractional Kelly under estimation error: **STAKE §4** (Chopra–Ziemba 1993 errors-in-means ~10× cost; MacLean–Ziemba–Blazenko 1992 growth-security; the `(2c−c²)` half-Kelly 75%/25% growth/variance trade-off, Thorp 2006). `λ` is **data-derived** (Bayesian shrinkage `λ ≈ 1/(1+CV²)` and/or the binding BRB drawdown constraint), not hand-set; half-Kelly is the prior, not the answer (STAKE §4.2–§4.3, Open Question 4).
- Concurrent matchdays: **STAKE §5** — the convex vector-Kelly program `max_f Σ_k π_k log((Of)_k)` s.t. `1ᵀf=1, f≥0` (cvxpy), with the renormalised-and-capped single-bet approximation as the deployable rule; independence of concurrent matches is the default to be **tested** (final-round dead-rubber / "biscotto" effects; STAKE §5.3, Open Question 2).
- Risk of ruin / minimum bankroll: **STAKE §6**. The Feller additive even-money form (eq. 7–8) is an even-money `b=1` sanity check only — the genuine `+b/−1` additive object is the adjustment-coefficient root, and the deployed dynamics are multiplicative; the **operational** number is the matchday-block bootstrap Monte-Carlo (STAKE §6.3), resampling *matchdays* (block = matchday; Politis–Romano 1994 stationary bootstrap) to preserve concurrency. `B` set by the precision target (`SE ≤ ε/10 ⇒ B ≥ 100(1−ε)/ε`; run `B = 10⁴`), not arbitrarily.
- Busseti–Ryu–Boyd drawdown bound: **STAKE §6.2** — `E[(rᵀb)^{−θ}] ≤ 1 ⟹ Prob(W_min<α) < α^θ`, `θ = log β_dd/log α_dd`; unconstrained full Kelly does **not** satisfy the `θ=1` bound (`Prob(W_min<0.7)≈0.40`); shrink until the constraint binds → fractional `λ < 1`. The drawdown target `(α_dd, β_dd)` maps to the deployed `λ`.
- Growth–drawdown frontier: **STAKE §7.3** — theory predicts RCK ≽ fractional-Kelly ≽ {fixed-fraction, level-to-odds}; deliver the frontier from the empirical bootstrap (F-07), reporting fractional-Kelly **and** the RCK solution so the gap is measured.
- **Transaction-cost / execution model (mandated by the quant reporting rule; specified here, ADR-stubbed in Phase 0 task 7.1).** The synthetic DNB is a **two-leg** position — stake `1/D` on the draw + `(D−1)/D` on the win (CALC §3.1, §5.1) — and the EV identity `o_DNB = W(D−1)/D` assumes **both legs fill at the quoted closing line simultaneously at zero slippage**. That idealization must be modeled and stressed, not assumed, before any net number:
  - **(i) Per-leg slippage**, calibrated from the **open→close move distribution** observed on the football-data Main leagues (Phase 1 task 9.1), per division-season × reference regime × odds bucket. The applied slippage is a **data-selected quantile of the absolute open→close move** (the §D.3 selection procedure — grid/empirical-quantile with a documented rationale and citation), **not a magic number**; it is applied to each leg's effective fill price in the `costs` config block.
  - **(ii) Leg-out assumption.** Default: **atomic two-leg fill at the closing line**, stated explicitly as an **idealization**, and **stress-tested by a one-tick adverse-move sensitivity** — re-price with the win leg (or draw leg) filling one tick worse after the other fills, to bound the cost of non-simultaneous execution (leg-out risk). Report the net metrics under both the atomic-fill idealization and the one-tick-adverse stress.
  - **(iii) Betfair-exchange commission as an effective overround.** The BFEC exchange fallback carries **2–5% commission on net winnings, not overround** (DATA §2.3). Convert the commission to an **effective overround** on the exchange DNB and **reconcile it against the synthetic-vs-quoted margin wedge `M_1X2 − M_AH`** (CALC §3.5, quantified in Phase 1 task 9), so the exchange-route cost is on the same footing as the quoted/synthetic-route cost.
  - **Net-of-cost is a precondition for reporting.** No ROI / Sharpe / growth number is reported as net until the `costs` block (slippage + leg-out + commission/effective-overround) is applied; gross numbers may be shown only alongside their net counterparts and explicitly labelled gross (Phase 4/5 acceptance). This is load-bearing for the honest verdict: the EV prior is already net-negative gross (A.2, CALC §8.2 worked −5.3%), and execution cost can only deepen it, so a "do not bet" verdict must be defended on **net-of-cost, executable** terms (gaps register cross-cutting observation 3).

### Phase 4 — Inference (bootstrap CIs, multiple-testing gate, deflated Sharpe, walk-forward)

**Objective.** Produce honest, selection-bias-corrected, serially-robust inference on the strategy: walk-forward out-of-sample estimation on leagues, the World Cup scored once as the held-out block, with bootstrap CIs, the multiple-testing gate, and the deflated Sharpe as the binding significance test.

**Inputs.** The walk-forward `Ledger` per scheme/cell (Phase 3); the frozen multiple-testing register and CV fold structure (Phase 0; methodology.md §2–§3); STAT; STAKE §7.2; EDGE §7.

**Methods (cited).**
- Pre-data power analysis is **frozen in Phase 0** (task 8) and only **re-run** here — Phase 4 evaluates the Phase-0-frozen `power_<HID>.md` calculation against the assembled `n`; it **never re-derives `δ` or `Var_DNB`** (doing so after the panel is seen is the Hoenig–Heisey 2001 contamination, STAT §6.1). The frozen record holds required-`n` `= κ·(z_{1−α/2}+z_{1−β})²σ_R²/δ²` with the LRV design-effect `κ = Ĵ/γ̂₀` for the mean/ROI test (**not** the Sharpe-annualisation `η(q)`) — **STAT §6**; the per-bet SD uses the three-point `Var_DNB`, not `o−1` (STAT §2.10; DATA §1.3; EDGE §5.2); the plausible edge `δ` is literature-anchored (EDGE §5.4, §7), not observed; **no post-hoc power** (Hoenig–Heisey 2001, [10.1198/000313001300339897](https://doi.org/10.1198/000313001300339897); STAT §6.1; `power-analysis` skill).
- Walk-forward, time-ordered, expanding-window CV on season boundaries; the World Cup is the final never-touched fold; threshold `τ`, `φ`, `c`, `λ` are selected purely out-of-fold — **STAT §9.1**; methodology.md §2; CALC §2.3. **The de-vig method is *not* in the per-fold selection set: it is frozen a priori in Phase 0 (Shin primary; power/basic as registered sensitivities; Phase 2 method, reconciliation paragraph), so it neither consumes a per-fold degree of freedom nor enters `K` / `N_eff` as a searched dimension.** The Phase-2 per-training-fold calibration diagnostic (reliability + Brier + RPS, World Cup and post-fold seasons excluded) is the pre-registered justification for that freeze, not an out-of-fold selection step — calibration gating and per-fold selection therefore do not both consume the same degree of freedom.
- Point-in-time leakage canary (`pit-canary` skill) run **before** any performance number is trusted: inject a future-knowing feature; if it does not dominate, the pipeline already leaks; confirm swapping opening↔closing odds changes the bet set — **STAT §9.2**; ARCH §3.1; DATA §5.3; EDGE Open Question 5.
- Performance metrics + bootstrap CIs: **STAT §2–§3**. Stationary bootstrap (Politis–Romano 1994) with Politis–White (2004) + Patton–Politis–White (2009) data-dependent block length (report `b̂` and `±50%` sensitivity, STAT Open Question 7); studentized bootstrap-`t` primary + BCa robustness; `B` from the Monte-Carlo-SE precision target (ARCH §5.6).
- HAC SEs: **STAT §5** — Andrews-QS (1991) primary, Newey–West (1994) Bartlett cross-check, with Andrews–Monahan prewhitening.
- Sharpe inference: per-bet Sharpe primary; Lo (2002) asymptotic SE; Opdyke (2007) non-IID distribution; the non-normality-robust SE (Bailey–López de Prado 2014 eq. 4 / Mertens 2002) with the *positive* per-bet skew sign argued from kurtosis — **STAT §2.3, §2.7–§2.8, §8.4**.
- Pairwise scheme comparison: Ledoit–Wolf (2008) studentized circular-block bootstrap on the *same* matches — **STAT §4**; drives T-11, F-15.
- Multiple-testing across the grid (the `multipletest-gate` skill enforces the register): Hansen (2005) SPA default / White (2000) Reality Check for "is there *any* edge"; Romano–Wolf (2005) for "which ones"; BH FDR exploratory — **STAT §7**; STAKE §7.2.
- Deflated / Probabilistic Sharpe + MinTRL: **STAT §8**; ARCH §6.3. `SR₀ = E[max SR]` over `N_eff` effective trials; the DSR denominator uses the *observed* `ŜR` in per-bet units; the denominator is **HAC-inflated** because the plain IID-non-normal DSR is anti-conservative under serial dependence, and the serially-robust (Opdyke/Ledoit–Wolf) check is binding when the two disagree (ARCH §6.3 serial-dependence caveat; STAT §8.2 effective-`N` Open Question 3).
- Closing-line value as the leading edge diagnostic **where defined**, computed from the *closing* synthetic-DNB line (never opening): **STAT §2.9**; EDGE §2.3, §7. CLV requires both an entry price and a closing price on the same selection, so it is **undefined on synthetic-only rows** (pre-2019/20 league + early-WC editions; Phase 1 task 10). A diagnostic undefined on a large fraction of the held-out sample cannot be the binding edge test there: CLV is therefore the primary diagnostic **only on the CLV-defined sub-sample**, and the **realized-return + DSR** test is the primary edge test on synthetic-only rows. Report the CLV-defined fraction (Phase 1 task 10) alongside the CLV estimate so the reader sees the support of the primary diagnostic.
- Assumption checks (decision-linked, not decorative): stationarity (ADF + KPSS + structural-break at 2018 VAR / 2026 format), independence (Ljung–Box, ACF/PACF, runs), distributional form (Jarque–Bera, report `γ̂₃, γ̂₄`), cross-sectional dependence (block on match-day) — **STAT §10**.

8. **Dependency pinning.** `requires-python` declared as a tight single-minor band (exact `==3.X.Y` in the lockfile), every direct dependency `==`-pinned in the manifest **and** the full transitive graph committed in `uv.lock`, native deps (BLAS, the chosen cvxpy solver backend) enumerated in an ADR, and `uv sync --frozen` proven on a clean CI runner — because the unpinned transitive surface (~13.5× expansion; 68.3% OOTB) is the dominant repro risk ([arXiv 2512.22387](https://arxiv.org/abs/2512.22387); Phase 0 task 1).
9. **Net-of-cost execution accounting.** The quant reporting rule's mandated transaction-cost model is specified as a `costs` config block + ledger cost hook (Phase 0 ADR 7.1; Phase 3 task 9): per-leg slippage selected from the open→close move distribution (Phase 1 task 9.1, §D.3-selected), the two-leg synthetic-DNB leg-out idealization stress-tested by a one-tick adverse move, and the BFEC commission converted to an effective overround reconciled with the `M_1X2 − M_AH` wedge (CALC §3.5; Phase 1 task 9). **No ROI/Sharpe/growth number is reported as net until the block is applied**, under both the atomic-fill idealization and the one-tick stress; gross is labelled gross and shown only alongside net (Phase 4/5 acceptance; gaps register transaction-cost row).
10. **Identity hygiene before any commit.** A fail-closed pre-commit/pre-push gate asserts `git config user.email`/`user.name` = the SKIE pseudonym (`useConfigOnly = true` confirmed) and scans staged files + notebook metadata for the OS username (`skoir`) / real email, blocking the first and every commit on a hit ([rules/publishing.md](../../../.claude/rules/publishing.md); Phase 0 task 2.1) — load-bearing because the venue is publication under the pseudonym.

## E. Phase dependency graph

```
Phase 0  (env + repro + pre-registration + register)
   │
   ▼
Phase 1  (data acquisition + validation)
   │
   ▼
Phase 2  (DNB construction + underdog labelling + 90-min settlement)
   │
   ▼
Phase 3  (staking grid + bankroll/ruin simulation)
   │
   ▼
Phase 4  (inference: bootstrap CIs + multiple-testing gate + DSR + walk-forward)   ◄── pre-data power-analysis FROZEN in Phase 0; Phase 4 only RE-RUNS it against assembled n
   │
   ▼
Phase 5  (deliverables: report card + figures + tables + Excel workbook)
```

Phases are strictly ordered (each consumes the prior phase's typed output). The pre-data `power-analysis` is **fully frozen in Phase 0** — `δ` and `Var_DNB` are sourced from the EDGE §5.2 analytic three-point form (literature-anchored, not observed) and written to `power_<HID>.md` with its SHA recorded in the ReproLog **before Phase 1 downloads any data**, so the analyst cannot calibrate the effect or the per-bet variance to the assembled panel's realized overround / draw rate `q` / DNB-variance distribution (the Hoenig–Heisey 2001 contamination, [10.1198/000313001300339897](https://doi.org/10.1198/000313001300339897); STAT §6.1). Phase 4 only **re-runs** the frozen calculation against the assembled `n` and verifies the `power_<HID>.md` SHA — it never re-derives `δ` or `Var_DNB`. Per-phase stage targets (Phase 0 task 9) allow each phase to be reproduced in isolation, each emitting its own ReproLog.
